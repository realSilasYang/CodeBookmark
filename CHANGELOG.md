# 📋 更新日志

[简体中文](./CHANGELOG.md) · [English](./CHANGELOG.en.md)

本项目的显著变化记录在此文件中，版本号遵循[语义化版本](https://semver.org/lang/zh-CN/)。新增版本时请使用[中文更新日志模板](https://github.com/realSilasYang/CodeBookmark/blob/main/docs/CHANGELOG_TEMPLATE.md)，从用户视角说明实际变化，并删除没有内容的分类。

## 🎉 版本 2.0.0 - 2026-07-23

### ✨ 新增

- **完整中英文本地化**：命令、菜单、设置、通知、日志、Webview、扩展清单和维护文档均提供中英文版本；中文 VS Code 默认显示中文，其他语言环境统一显示英文。
- **多协议 AI 地址引擎**：AI 地址支持资源 Endpoint、API Base URL 和完整请求 URL，可自动识别 OpenAI Responses、Chat Completions、Azure OpenAI、Anthropic Messages、Gemini 原生或兼容接口以及 Ollama。
- **AI 语义图标体系**：从 1499 个图标资源中精选 99 个高辨识度语义概念，为每个概念设置独立证据、冲突和优先级规则；只有语义明确时才使用专用图标。
- **最近图标同步**：最近使用的图标加入 VS Code 设置同步范围，在使用同一账号的设备间保留图标选择习惯。

---

### 🚀 优化

- **上下文 AI 菜单**：根据是否打开文件夹、当前脚本是否存在、脚本和文件夹内是否已有书签动态显示生成与优化功能；仅剩一个操作时自动取消冗余子菜单。
- **AI 配置体验**：配置不完整时自动打开 AI 设置；连接验证成功后把实际可用的完整地址写回原配置层级，失败或取消时保持原值。
- **AI 请求可靠性**：保留用户明确选择的协议，支持同源兼容路由回退、远程 HTTP 发送确认、请求总超时、响应状态分类，以及工作区或源码变化后的过期结果拒绝。
- **操作结果统计**：生成、优化、删除、导入等完成提示统一展示书签总数与各层级数量，批量操作同时说明成功、跳过和失败范围。
- **运行时与发布包**：将 113 个运行时模块打包为单一入口，减少 VSIX 中的 JavaScript 文件数量，同时保留全部 1499 个图标和四份本地化清单。
- **发布与维护文档**：更新日志、发布指南、贡献、安全、支持和 Pull Request 文档改为结构化中英文版本；GitHub Release 直接使用对应版本的中文更新日志。

---

### 🐛 修复

- **AI 地址补全**：修复 Azure `/openai/v1/` 被错误追加固定路由、Gemini OpenAI 兼容地址重复追加 `/chat`，以及半完整请求路径无法准确补全的问题。
- **AI 图标误配**：修复仅凭宽泛关键词强行选择无关图标的问题；同优先级语义冲突或证据不足时稳定回退默认图标。
- **扩展停用持久化**：修复停用阶段重复写入未变化的撤销状态，并避免 VS Code 已关闭输出通道时产生二次异常。
- **本地化行为边界**：命令 ID、设置键、持久化字段、Webview 消息和 AI 协议保持语言无关，防止翻译文本改变功能分支或数据兼容性。

---

### ⚠️ 重要说明

- **AI 配置键升级**：2.0.0 不再读取旧键 `codebookmark.AI.endpoint` 和 `codebookmark.AI.apiKey`。升级后请在设置中重新填写 `codebookmark.AI.address` 与 `codebookmark.AI.APIKey`；API Key 仍以明文保存在所选 VS Code 设置层级中。

## 🎉 版本 1.1.0 - 2026-07-22

### ✨ 新增

- **书签配置管理**：新增书签配置管理界面，统一查看脚本配置、工作区排序记录和存储迁移记录。
- **配置统计信息**：展示书签总数、分级数量、自动或异常数量，以及迁移来源、目标、时间和文件统计。
- **配置安全清理**：支持按存储记录安全清理配置和历史元数据，自动防止路径穿越、文件版本变化和同名记录冲突。

---

### 🚀 优化

- **配置记录身份**：书签配置管理改用存储相对路径作为记录身份，工作区排序记录不再因文件名相同而相互覆盖。
- **删除确认界面**：删除确认改为适配 VS Code 深色、浅色和高对比度主题的页面内弹窗，并保留影响说明、焦点管理和二次确认。
- **存储目录迁移**：迁移成功后清理来源目录中的 CodeBookmark 数据，同时保留迁移结果记录供管理界面查看。

---

### 🐛 修复

- **历史记录管理**：修复工作区排序记录和存储迁移记录无法查看或清理的问题。
- **书签数量统计**：修复配置管理界面将历史元数据误计入书签总数的问题。

## 🎉 版本 1.0.0 - 2026-07-21

### ✨ 新增

- **分层书签树**：支持拖拽、嵌套和排序的分层代码书签树。
- **全局脚本身份**：工作区与独立脚本共享全局脚本身份和持久化结构。
- **移动自动追随**：文件、目录和工作区根目录移动后自动恢复书签绑定。
- **注释自动书签**：支持 TODO、FIXME、BUG 注释自动书签及语言配置发现。
- **多格式导出**：支持 Markdown、HTML、CSV、纯文本和配置源文件导出，以及按源文件批量导出。
- **完整书签工作流**：支持单文件或工作区配置导入、撤销重做、图标选择和行内标签。
- **AI 辅助能力**：支持配置接口地址、模型与 API Key，用 AI 生成书签、优化标签和选择语义图标。
- **可靠持久化**：支持原子保存、外部配置变更合并、存储根目录转移和异常恢复日志。
- **质量保障**：建立 TypeScript、ESLint、专项验证和 VS Code Extension Host 集成测试。
- **开源发布设施**：提供 GitHub Issue 与 Pull Request 模板、依赖更新、CI 打包校验和标签驱动的 VSIX Release 流程。
