# CodeBookmark（代码书签）使用说明

CodeBookmark 是一款专注于代码段标记与结构化管理的 VS Code 扩展。底层基于非阻塞并发 I/O 构建，支持大批量书签数据的毫秒级加载，提供零卡顿的代码导航体验。

## 一、快捷键操作

| 快捷键 / 按键 | 生效范围 | 功能操作 |
| :--- | :--- | :--- |
| `Ctrl+B` | 编辑器 | 切换书签（在当前光标行添加/移除） |
| `Alt+B` | 全局 | 唤起/聚焦书签面板 |
| `Ctrl+Alt+Shift+B` | 编辑器 | 强制添加书签（忽略该行已有状态） |
| `Ctrl+Alt+Shift+D` | 编辑器 | 强制删除书签 |
| `F2` | 书签面板 | 重命名当前选中的书签或文件夹 |
| `Delete` | 书签面板 | 删除当前选中的节点（支持多选） |

## 二、核心操作指南

### 1. 节点管理
*   **拖拽排版**：在面板中拖拽节点，自由调整先后顺序或嵌套层级。
*   **属性修改**：右键点击节点，更改显示图标或重新绑定行号锚点。
*   **视图切换**：点击面板顶部菜单，切换物理行号、字母名称、文件路径或自定义拖拽视图。

### 2. 撤销恢复
*   **无级回滚**：书签增删改查、拖拽排序等所有行为均支持撤销与恢复。
*   **触发方式**：点击面板顶部的 `↩️` / `↪️` 按钮（悬停按钮可预览即将撤销的指令）。

### 3. 数据存储
*   **工作区模式（默认）**：打开工作区时，书签独立保存在该项目的本地沙盒中。
*   **全局模式**：打开单文件时，书签平铺保存在设置项 `globalStoragePath` 对应的路径。

## 三、AI 智能辅助

使用 AI 能力前，请在 VS Code 设置 (`codebookmark.ai`) 中完成基础配置：`Endpoint`（API 地址）、`Model`（模型名称）以及 `API Key`。

配置完成后，点击书签面板右上角的 **⚡** (闪电) 按钮，系统将提供三种智能策略：
*   **追加到当前文件的书签**：让大模型深度阅读当前代码，提炼核心逻辑并追加为全新的智能书签，与您的手工书签和谐共处。
*   **覆盖当前文件的书签**：在提取最新书签落盘前，自动且彻底地清空当前文件中的所有旧书签，焕然一新。
*   **优化当前文件的书签标签**：不对节点层级、顺序和位置做任何破坏性修改，仅借助大模型上下文理解能力，原地将已有书签重命名为更为洗练、精准的极客式短语。


## 四、项目结构

本作遵循模块化架构设计，核心代码集结于 `src` 目录之下：

*   **`extension.ts`**：扩展的主入口文件，负责生命周期的管理及核心组件的初始化。
*   **`commands/`**：VS Code 命令总线，统筹所有的快捷操作及右键菜单指令分发。
*   **`models/`**：核心数据结构与视图实体（如 `Bookmark`、`BookmarkSet` 等）。
*   **`providers/`**：视图解析层，包含驱动左侧资源树状态更新的 `TreeDataProvider` 实现。
*   **`repository/`**：本地存储引擎，专门负责大批量书签数据的异步读写与落盘。
*   **`subscriptions/`**：文本编辑器事件监听模块（如行号变更、文件内容重写时的动态装饰更新）。
*   **`util/`**：基建与工具链，涵盖文件路径换算、AI 大模型通信服务 (`AIService`) 以及各类常量枚举。
