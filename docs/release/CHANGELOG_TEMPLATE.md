# 📝 中文更新日志模板

[简体中文](./CHANGELOG_TEMPLATE.md) · [English](./CHANGELOG_TEMPLATE.en.md)

新增版本时，将下面的版本块复制到 `CHANGELOG.md` 顶端语言链接之后。只保留本版本实际包含的分类；每项使用“**功能名称**：面向用户的变化与收益”格式，不要直接罗列提交信息、内部类名或无关实现细节。

## 🎉 版本 X.Y.Z - YYYY-MM-DD

### ⚠️ 重要说明

- **升级提醒**：仅在存在配置迁移、不兼容变化或需要用户操作时保留本节，并给出明确处理方法。

---

### ✨ 新增

- **功能名称**：说明新增了什么、适用于什么场景，以及用户可以获得什么。

---

### 🚀 优化

- **功能名称**：说明原有体验如何改善，以及用户能够感知的具体变化。

---

### 🐛 修复

- **问题名称**：说明修复了什么可观察问题，以及修复后的正确行为。

## 📐 编写规则

- 版本标题固定使用 `🎉 版本 X.Y.Z - YYYY-MM-DD`，版本号必须与扩展清单一致。
- 常规分类固定为 `✨ 新增`、`🚀 优化`、`🐛 修复`；没有内容的分类直接删除。
- `⚠️ 重要说明` 只用于升级操作、兼容性变化或其他必须提前告知用户的事项；存在时必须放在版本内容首位。
- 每项先用粗体短语概括功能或问题，再用一句完整中文说明实际影响。
- 合并同一功能的零散提交，避免使用“重构某文件”“调整某变量”等用户无法理解的表述。
- 一级总标题和各级结构标题均保留对应 Emoji，确保 GitHub Release 与 `CHANGELOG.md` 易于扫描。
